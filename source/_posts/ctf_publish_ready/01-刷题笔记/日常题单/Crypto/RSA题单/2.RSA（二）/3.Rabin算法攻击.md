# 三：解题思路：

和上题一样，我们会发现本题的e非常小，有e=2，但我们会发现c不是一个完全平方数，则说明不能用低加密指数进行攻击的方法解决。
同时我们发现本题的e和phi不互素，因为e和phi都是偶数，故我们也得不到RSA的私钥。实际上，除了RSA算法外，还有其他很多的非对称加密方案，例如我们在本题中实际上使用了Rabing算法进行加密，只是它类似于RSA，故在CTF中我们将其划分到RSA的攻击范围中，在后续的学习中我们还会遇到很多像RSA但实际不是RSA的加密算法。
在Rabin算法中加密流程为：

1. 取两个大素数（P，Q）满足：
   $$
   p\equiv q\equiv3(mod4),有n=pq
   $$

   这也是为什么本题有个断言p%4=3andq%4=3
2. 加密过程为;
   $$
   c\equiv m^3(modn)
   $$

我们会发现其实这就是本题的加密过程，同时它也等于e=2时的RSA加密过程。
对于Rabin的解密，稍微复杂一些，这里我们需要了解二次剩余和欧拉准则，关于Rabin算法的详细推导可见
由此我们可知Rabin算法适用于这种e=2，p,q已知且满足%4=3的题目中。此时我们可以通过Rabin算法解得四个明文，再一一测试即可得到最终正确明文

from Crypto.Util.number import *
from gmpy2 import *

p = 67711062621608175960173275013534737889372437946924512522469843485353704013203
q = 91200252033239924238625443698357031288749612243099728355449192607988117291739
e = 2
c = 5251890478898826530186837207902117236305266861227697352434308106457554098811792713226801824100629792962861125855696719512180887415808454466978721678349614

def rabin_attack(c, n, p, q):
    c1 = powmod(c, (p+1)//4, p)
    c2 = powmod(c, (q+1)//4, q)
    cp1 = p - c1
    cp2 = q - c2
    t1 = invert(p, q)
    t2 = invert(q, p)
    m1 = (q*c1*c2 + p*c2*t1) % n
    m2 = (q*c1*t2 + p*cp2*t1) % n
    m3 = (q*cp1*t2 + p*c2*t1) % n
    m4 = (q*cp1*t2 + p*cp2*t1) % n
    return m1, m2, m3, m4
ms = rabin_attack(c, p*q, p, q)
for m in ms:
    print(long_to_bytes(m))
